# 系统设计文档

## 1. 总体架构

系统同时提供命令行程序和图形界面程序，核心备份逻辑复用同一套业务接口：

- 命令行接口层：解析 `backup`、`restore`、`list` 命令及参数。
- 图形界面层：提供备份、还原、归档查看页签，负责用户输入、文件选择和状态提示。
- 业务逻辑层：完成目录扫描、筛选、备份、还原、元数据处理。
- 归档格式层：负责 `.sdb` 文件的二进制读写、压缩、加密和校验。
- 文件系统适配层：使用 Win32 API 完成目录遍历、目录创建、文件时间和属性处理。

主要源文件：

- `src/main.cpp`：命令行入口和参数解析。
- `src/gui.cpp`：Win32 图形界面入口和控件事件处理。
- `src/backup.hpp`：业务接口和数据结构定义。
- `src/backup.cpp`：归档格式、备份还原算法实现。

## 2. 模块设计

### 2.1 命令行模块

支持三个命令：

- `backup <source_dir> <archive.sdb>`：创建备份归档。
- `restore <archive.sdb> <target_dir>`：恢复备份归档。
- `list <archive.sdb>`：查看归档内容。

该模块负责参数合法性检查，并将选项转换为 `BackupOptions` 或 `RestoreOptions`。

### 2.2 图形界面模块

图形界面使用 Win32 原生控件实现，不依赖额外 GUI 框架。主窗口包含三个页签：

- Backup：选择源目录、归档文件，设置压缩、密码和筛选条件，执行备份。
- Restore：选择归档文件、目标目录，设置密码和覆盖选项，执行还原。
- Archive List：选择归档文件，使用表格展示条目类型、原始大小、存储大小、修改时间、压缩/加密标记和路径。

界面层只负责输入收集和结果展示，备份、还原、归档查看均调用 `backup.hpp` 暴露的业务接口。

### 2.3 文件扫描与筛选模块

扫描源目录时使用 Win32 `FindFirstFileA` / `FindNextFileA` 递归遍历。目录条目总是保存，普通文件根据筛选条件决定是否保存。遍历时跳过 reparse point，避免目录链接造成循环。

筛选条件包括：

- `--include-ext`
- `--exclude-ext`
- `--name-contains`
- `--min-size`
- `--max-size`
- `--modified-after`

### 2.4 归档读写模块

归档文件由文件头和若干条目组成。所有整数以小端序保存，避免结构体内存布局差异。

文件头字段：

- magic：`SDBK2026`
- version：格式版本
- flags：归档级标记
- createdAt：创建时间
- passwordFingerprint：密码指纹
- entryCount：条目数量

条目字段：

- type：目录或文件
- flags：压缩、加密等条目标记
- path：相对路径
- originalSize：原始大小
- storedSize：归档中载荷大小
- modifiedTime：修改时间
- permissions：权限位
- checksum：原始数据 FNV-1a 校验和
- payload：文件载荷，目录无载荷

### 2.5 压缩模块

压缩算法为内置 RLE。备份时先尝试压缩，只有压缩结果小于原始数据时才启用压缩标记。还原时根据条目标记自动解压。

### 2.6 加密模块

用户提供密码时，文件载荷在压缩之后进行加密。实现使用基于 SplitMix64 伪随机序列的异或流，加密和解密过程相同。归档头保存密码指纹，用于在还原前检测错误密码。

### 2.7 元数据模块

备份时保存 Windows 文件属性和最后修改时间。还原文件后立即恢复文件元数据；目录元数据在所有文件写入完成后按路径长度逆序恢复，避免创建子文件时改变父目录时间。

## 3. 安全与健壮性设计

- 备份文件不能放在源目录内部，避免备份过程中递归包含自身。
- 还原时拒绝绝对路径、空路径、`.` 和 `..` 路径，防止写出目标目录。
- 还原默认不覆盖已有文件，需要显式指定 `--overwrite`。
- 文件载荷恢复后计算校验和，发现损坏归档或错误密码时停止。

## 4. 后续可扩展点

- 增量备份：为条目增加文件版本和块级校验。
- 实时备份：通过文件系统事件监控触发备份。
- 网络备份：将归档上传到远程服务并增加用户管理。
- GUI 美化：可继续增加进度条、任务取消和操作历史。
