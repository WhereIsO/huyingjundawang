# 需求分析说明书

## 1. 项目背景

本项目依据《软件开发综合实验》要求，实现一款基本可用的数据备份软件。软件面向个人用户或小型项目组，用于将指定目录树中的文件保存到指定位置，并在需要时恢复到目标目录。

## 2. 用户角色

- 普通用户：执行日常目录备份和数据还原。
- 项目维护者：构建、测试、维护软件，并根据需要扩展备份策略。

## 3. 功能需求

### 3.1 基础功能

- 数据备份：用户指定源目录和归档文件路径，软件递归扫描目录树并保存目录、文件内容和空目录。
- 数据还原：用户指定归档文件和目标目录，软件恢复目录结构和文件内容。
- 归档查看：用户可查看归档内的条目、大小、修改时间和扩展功能标记。

### 3.2 扩展功能

- 元数据支持：保存文件权限和修改时间，还原时尽量恢复。
- 自定义备份：支持按扩展名包含、扩展名排除、文件名包含、最小大小、最大大小、修改时间筛选。
- 打包解包：所有备份条目写入单个 `.sdb` 文件，方便移动和保存。
- 压缩解压：支持内置 RLE 压缩算法，压缩有效时保存压缩数据。
- 加密解密：用户可指定密码，软件对文件载荷加密保存，还原时校验密码。
- 图形界面：提供备份、还原、归档查看三个操作页签，并支持目录选择、文件选择、筛选条件输入和结果状态提示。
- 健壮性保护：还原时检查归档内路径，禁止绝对路径和 `..` 路径逃逸。

## 4. 非功能需求

- 正确性：还原后的文件内容必须与备份时一致。
- 易用性：用户既可通过命令行参数完成备份、还原、查看，也可使用图形界面完成常见操作。
- 健壮性：非法参数、错误密码、损坏归档、目标文件已存在等情况应给出错误提示。
- 可移植性：核心业务逻辑基于 C++17，文件系统访问层适配 Windows/MinGW 环境。
- 可维护性：备份逻辑、归档格式和命令行入口分离，便于后续扩展。

## 5. 约束

- 当前版本处理普通文件和目录，不主动跟随或恢复符号链接、管道、设备文件。
- 当前版本以 Windows 为主要运行环境，Linux/MacOS 需要替换文件系统访问层。
- 加密功能用于课程实验场景，不声明达到专业密码学安全等级。
- RLE 压缩适合重复字节较多的文件，对随机数据可能不会生效。

## 6. 验收标准

- 可成功备份包含多级目录、普通文件、空目录的源目录。
- 可成功还原到新目录，文件内容一致。
- 使用 `--exclude-ext` 等筛选参数时，符合条件的文件被排除或包含。
- 使用 `--password` 生成的归档必须提供正确密码才能还原。
- `bin/sdb_gui.exe` 可正常构建，图形界面提供备份、还原、查看入口。
- `mingw32-make test` 通过端到端测试。
