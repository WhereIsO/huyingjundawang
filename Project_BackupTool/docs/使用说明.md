## 2. 运行真实后端

默认直接启动 `BackupTool.exe` 时会使用 `MockBackend`，用于界面演示。要使用真实备份/还原功能，必须设置环境变量：

```powershell
$env:BACKUP_BACKEND_MODE = "real"
$env:Path = "G:\Anaconda3\Library\bin;$env:Path"
$env:QT_QPA_PLATFORM_PLUGIN_PATH = "G:\Anaconda3\Library\plugins\platforms"
& "G:\_build_backup_tool_gui_ascii\BackupTool.exe"
```

说明：

- `BACKUP_BACKEND_MODE=real`：让工厂函数返回 `RealBackend`。
- `G:\Anaconda3\Library\bin`：提供 Qt5Widgets、Qt5Core、Qt5Concurrent 等运行库。
- `QT_QPA_PLATFORM_PLUGIN_PATH`：提供 Windows 平台插件 `qwindows.dll`。如果双击能正常打开，可以不手动设置。

## 4. 图形界面使用流程

### 4.1 备份

1. 按第 2 节方式启动真实后端。
2. 打开 `① 备份` 标签页。
3. 在“源目录”中选择要备份的文件夹。
4. 在“输出备份包”中选择或填写 `.pbackup` 文件路径。
5. 默认可勾选“哈夫曼压缩”。
6. 如需加密，勾选 AES-256 加密并输入密码。
7. 如需筛选，先切换到 `② 筛选` 标签页填写条件。
8. 回到 `① 备份`，点击“开始备份”。
9. 底部进度条和日志面板会显示 `Scanning / Filtering / Writing / Done` 等阶段。

备份成功后会生成单个 `.pbackup` 文件。该文件包含索引、元数据、payload、CRC32 与 SHA-256 校验信息。

### 4.2 筛选

筛选页支持 6 类条件。留空表示不过滤，多个非空条件之间按 AND 逻辑同时生效。

| 筛选类别 | 字段 | 示例 |
|---|---|---|
| 路径 | includePath / excludePath | 包含 `docs`，排除 `tmp` |
| 类型 | typeFilter | `file,dir,symlink,hardlink,junction,reparse` |
| 名称 | nameGlob | `*.docx`、`*.txt` |
| 时间 | mtimeAfter / mtimeBefore | `2026-07-06` |
| 尺寸 | sizeMin / sizeMax | `1024`、`10485760` |
| 用户 | ownerSid | `S-1-5-21-...` |

注意：ownerSid 是 Windows 所有者 SID，普通用户不一定知道，可留空。

### 4.3 还原

1. 打开 `③ 还原` 标签页。
2. 选择 `.pbackup` 备份包。
3. 选择还原目标目录。
4. 如果备份包加密，输入正确密码。
5. 如果允许覆盖同名文件，勾选“覆盖已存在文件”。
6. 点击“开始还原”。
7. 日志面板显示还原过程和可能的警告。

错误密码、损坏包、目标文件已存在且未勾选覆盖等情况会失败并显示错误。

## 5. 已支持功能

- 基本备份：目录树写入单个 `.pbackup` 文件。
- 基本还原：从 `.pbackup` 还原到指定目录。
- 文件类型：普通文件、目录、空目录、符号链接、硬链接、Junction；未知 ReparsePoint 尽力恢复为目录并记录 warning。
- 元数据：大小、Windows 属性、创建/访问/修改时间、Owner SID、ACL 摘要。
- 自定义筛选：路径、类型、名称、时间、尺寸、用户。
- 打包解包：自定义 `.pbackup` 二进制格式。
- 压缩解压：自研哈夫曼压缩，不使用第三方压缩库。
- 加密解密：Windows CNG AES-256-GCM，PBKDF2-SHA256 派生密钥。
- GUI：Qt 5 Widgets，进度条、日志、备份页、筛选页、还原页。
- 自动化测试：CTest 43 项通过。

## 6. 注意事项

1. **真实后端需要环境变量**：未设置 `BACKUP_BACKEND_MODE=real` 时默认使用 Mock 后端。
2. **符号链接需要权限**：Windows 创建 symlink 可能需要开发者模式或管理员权限。测试环境无权限时 symlink 用例会跳过。
3. **Junction 已支持真实恢复**：Junction 扫描和备份还原已有自动化测试覆盖。
4. **中文路径构建问题已规避**：使用 `build_msvc.bat` 的 ASCII 镜像路径构建。
5. **Qt 运行库问题**：若启动时报 Qt DLL 或 platform plugin 缺失，按第 2 节设置 `Path` 和 `QT_QPA_PLATFORM_PLUGIN_PATH`。
6. **密码不可恢复**：加密包忘记密码无法还原；错误密码会被 AES-GCM 认证失败拦截。

## 7. 演示建议

建议准备一个样例目录：

```text
demo_src/
  a.txt
  docs/report.txt
  empty_dir/
  keep.log
```

演示顺序：

1. 启动真实后端。
2. 使用默认压缩备份 `demo_src` 到 `demo.pbackup`。
3. 清空或新建 `demo_restore` 目录。
4. 还原 `demo.pbackup` 到 `demo_restore`。
5. 对比 `a.txt` 和 `docs/report.txt` 内容一致。
6. 再演示加密备份，故意输入错误密码还原，展示失败提示。
