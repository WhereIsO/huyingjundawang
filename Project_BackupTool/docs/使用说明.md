# 数据备份软件使用说明

本文档记录当前工程的真实运行方式。当前项目已经包含自带运行库的 `bin` 目录和可分发压缩包，不需要依赖某台电脑上的 Anaconda、Qt 安装目录或固定盘符。

## 1. 当前可运行版本

推荐入口：

```text
Project_BackupTool\启动软件-本地.bat
```

该脚本会设置 `BACKUP_BACKEND_MODE=real`，并从 `Project_BackupTool\bin` 启动程序。`bin` 目录内已经包含 `BackupTool.exe`、Qt DLL、VC 运行库和 `platforms/qwindows.dll`。

演示模式入口：

```text
Project_BackupTool\启动软件-仅演示界面.bat
```

演示模式使用 `MockBackend`，只展示界面、日志和进度，不进行真实备份/还原。

可分发包：

```text
Project_BackupTool\dist\数据备份工具_可分发.zip
```

解压后双击其中的 `启动-数据备份工具.bat` 即可进入真实模式。

## 2. 构建与测试

源码目录：

```text
Project_BackupTool\gui\BackupTool
```

推荐构建：

```bat
cd gui\BackupTool
build_msvc.bat
```

`build_msvc.bat` 不再写死本机路径。可选环境变量：

| 变量 | 用途 |
|---|---|
| `VCVARS` | 指向 `vcvars64.bat`；不设置时脚本会尝试查找 VS 2022 常见安装路径 |
| `QT_PREFIX` | Qt 的 CMake 前缀，如 `C:\Qt\5.15.2\msvc2019_64` |
| `ASCII_SRC` | ASCII 镜像源码目录；默认 `%TEMP%\BackupTool_src` |
| `BUILD` | 构建目录；默认 `%TEMP%\BackupTool_build` |
| `GENERATOR` | CMake generator；默认 `Ninja` |
| `RUN_TESTS` | 是否运行 CTest，默认 `1` |
| `BACKUP_ALLOW_GTEST_DOWNLOAD` | 是否允许 CMake 下载 GoogleTest，默认 `OFF` |

说明：

- 普通运行不需要安装 Qt，因为 `bin` 已自带运行库。
- 从源码重新构建仍需要与 Qt ABI 匹配的编译器和 Qt 开发包。
- CMake 默认不联网下载 GoogleTest；本机没有 GTest 时会跳过测试目标。需要自动下载时显式设置 `BACKUP_ALLOW_GTEST_DOWNLOAD=ON`。

## 3. 图形界面使用流程

### 3.1 备份

1. 双击 `启动软件-本地.bat`。
2. 打开 `① 备份` 标签页。
3. 选择源目录。
4. 选择或填写 `.pbackup` 备份包输出路径。
5. 按需勾选哈夫曼压缩和 AES-256 加密。
6. 如需筛选，切到 `② 筛选` 设置条件，再回到备份页。
7. 点击“开始备份”。

### 3.2 筛选

筛选页支持路径、名称、类型、大小、修改时间、所有者 SID 六类条件。默认不启用类型筛选，因此空目录、Junction、未知 ReparsePoint 不会被默认排除。多个非空条件之间按 AND 生效。

### 3.3 还原

1. 打开 `③ 还原` 标签页。
2. 选择 `.pbackup` 备份包。
3. 选择还原目标目录。
4. 加密包需要输入正确密码。
5. 如需覆盖同名文件，勾选“覆盖已存在的文件”。
6. 点击“开始还原”。

还原时会校验包格式、CRC32、SHA-256 和 AES-GCM 认证标签。包内路径若包含绝对路径或 `..` 越界路径会被拒绝。

## 4. 已支持功能

- 基本备份与还原。
- 单文件 `.pbackup` 归档。
- 自研哈夫曼压缩。
- Windows CNG AES-256-GCM 加密，PBKDF2-SHA256 派生密钥。
- 路径、名称、类型、大小、时间、用户六类筛选。
- 普通文件、目录、空目录、符号链接、硬链接、Junction、未知 ReparsePoint 识别与恢复策略。
- 元数据采集与尽力恢复。
- Qt 5 Widgets 图形界面、进度条、分级日志。
- GoogleTest / CTest 自动化测试；当前测试源文件中共有 44 项 `TEST(...)`，其中新增路径穿越防护用例。

## 5. 注意事项

1. 真实模式请使用 `启动软件-本地.bat` 或分发包中的 `启动-数据备份工具.bat`。
2. 不要把 `BackupTool.exe` 单独拷走运行；需要保持 `bin` 目录内 DLL 和插件目录完整。
3. Windows 创建符号链接可能需要开发者模式或管理员权限。
4. 加密密码不可恢复，忘记密码无法还原。
5. 从源码构建时，Qt 的 ABI 必须与编译器匹配。

## 6. 演示建议

准备一个小样例目录：

```text
demo_src/
  a.txt
  docs/report.txt
  empty_dir/
  keep.log
```

演示顺序：

1. 真实模式启动。
2. 默认压缩备份 `demo_src` 到 `demo.pbackup`。
3. 新建 `demo_restore` 目录并还原。
4. 对比文件内容一致。
5. 再演示加密备份，并故意输入错误密码展示失败提示。
