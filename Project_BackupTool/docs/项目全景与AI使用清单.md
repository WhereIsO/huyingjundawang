## 一、一句话概括

这是一个 Windows 桌面**数据备份软件**：能把一个文件夹打包成单个 `.pbackup` 文件（可压缩、可加密），也能从这个文件还原回来。
前端界面用 Qt 5 写，后端核心用纯 C++ 写，两者已经接通。历史验证为 43 项测试全过；当前源码新增路径穿越防护测试后共有 44 项测试源码，需重新跑 CTest 更新最终截图。

## 六、答辩话术准备（老师大概率会问）

**Q：你们这个软件用 AI 写了多少？自己做了什么？**
A：核心代码由 AI 辅助生成，但功能选型、接口设计、真机验证、测试审阅和演示都是我们完成的。我们采用“前后端契约先行”的方法组织开发：先定义 `BackendAdapter` 接口，前端和后端分别实现，最后接通，并用自动化测试验证核心路径。

**Q：为什么用 Qt / MSVC？**
A：Qt 是课程示例采用的 GUI 框架；MSVC 是因为本机 Qt 是 MSVC 编译的，必须用同一编译器（ABI 兼容），这也是我们踩过的一个真实工程坑。

**Q：压缩和加密是自己写的吗？**
A：压缩（哈夫曼）是自研的，符合课程“不能用第三方压缩库”的要求；加密用 Windows 系统的 CNG（AES-256-GCM），课程允许加密调用成熟密码学 API，禁止的是手写加密算法。

**Q：演示一下？**
A：按 `组员使用手册` 第四节的流程——备份一个文件夹 → 还原 → 对比一致 → 加密备份 + 故意输错密码展示拦截。
